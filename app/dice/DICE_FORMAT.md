# Dice Response Format

Правила форматирования ответа бота для команды `/roll`.

## Структура ответа

```
Кидаю {expression}
> -> {step_1}
> -> {step_2}
> ...
> = {total}
```

Строки `->` — промежуточные шаги вычисления. Их количество зависит от сложности выражения.

---

## Правила отображения шагов

### Правило 1: `= subtotal` только в промежуточных шагах

`= subtotal` в конце строки `->` появляется **только если над этим узлом есть ещё операции**.
На самом нижнем уровне дерева (финальная строка `->`) `= subtotal` не показывается.

```
d20          →  > -> [**9**]       ← нет = 9
                > = 9

2*(3d10)     →  > -> [**2**]+[**7**]+[**9**] = 18   ← есть = 18, т.к. выше умножение
                > -> 2 * 18
                > = 36
```

### Правило 2: отдельная строка только для 2+ кубов

Субвыражение получает **собственную строку** `->` только если в нём **2 или более** кубиков.
Один куб — всегда инлайн, вне зависимости от контекста.

```
2*(3d10)       →  отдельная строка для 3d10 (3 куба)
2*d6           →  d6 инлайн: > -> 2 * [**4**]
2*(d12+4+5)    →  d12 инлайн: > -> 2 * ([**8**] + 4 + 5)
(3d6)*2+(1d8)*3 → отдельная строка для 3d6, но 1d8 инлайн
```

### Правило 3: чистая арифметика без кубов

Никаких строк `->`. Только финальный результат.

```
2+3*4   →   > = 14
```

### Правило 4: глубокая вложенность

Показываем все промежуточные уровни.

```
2*(3*(d6))   →   > -> [**4**] = 4
                 > -> 3 * 4 = 12
                 > -> 2 * 12
                 > = 24
```
*(Поведение может быть скорректировано после тестирования на практике.)*

## Примеры

### Простые случаи

```
d20            →   > -> [**9**]
                   > = 9

3d6            →   > -> [**1**] + [**4**] + [**6**]
                   > = 11

3d6+5          →   > -> [**1**] + [**4**] + [**6**] + 5
                   > = 16
```

### С умножением

```
2*(3d10)       →   > -> [**2**] + [**7**] + [**9**] = 18
                   > -> 2 * 18
                   > = 36

2*d6           →   > -> 2 * [**4**]
                   > = 8

(3d6+1d8)*2    →   > -> [**1**]+[**4**]+[**6**] + [**7**] = 18
                   > -> 2 * 18
                   > = 36
```

### Несколько групп

```
(3d6)*2+(1d8)*3  →  > -> [**1**]+[**4**]+[**6**] = 11
                    > -> 2*11 + [**7**]*3
                    > = 43
```
*1d8 инлайн (правило 2), 3d6 — отдельная строка.*

### Преимущество / помеха

```
ad20+5         →   > -> a[**17**|~~3~~] + 5
                   > = 22

dd20           →   > -> d[**8**|~~15~~]
                   > = 8
```

## Поля RollResult для форматтера

Форматтер опирается на три поля каждого узла в дереве:

| Поле | Описание | Пример для `3d6=[1,4,6]` |
|------|----------|--------------------------|
| `dice_trace` | Все кубики раскрыты, операции включены | `"[**1**] + [**4**] + [**6**]"` |
| `expr_trace` | Числовое представление для родительского узла | `"11"` |
| `dice_count` | Число кубиков в субвыражении | `3` |

Правило показа отдельной строки: `dice_count >= 2` AND узел не является корнем.
